# User Stories

| ID | As a… | I want to… | So that… |
|----|-------|-----------|----------|
| US-1 | Developer | Compare a single-model fighter vs a two-step pipeline fighter on my real inputs | I can decide if chaining prompts actually improves results |
| US-2 | AI engineer | Test GPT-4o-mini alone vs Llama→GPT-4o pipeline on cost × quality | I can justify the architecture tradeoff to my team |

| US-5 | Any user | Upload markdown files as sources | I can test with my actual documentation or prompts |

| US-7 | Any user | Configure temperature and tools per step | I can test OpenAI web_search in step 1 vs not using it |
| US-8 | Any user | Watch live results appear step by step | I'm not staring at a blank screen during a long run |
| US-9 | Any user | Edit the judge rubric | I can score based on my domain's definition of "good" |
| US-10 | Power user | Enter a custom model ID not in the catalog | I can test new models as soon as providers ship them |
| US-11 | Any user | See a markdown report summarising fighter rankings | I can share findings with a colleague by copy-pasting the markdown |
| US-12 | Any user | Drill down into each step's input/output | I can debug which step in a pipeline caused a poor result |

| US-14 | Developer | See credit costs alongside token costs in results | I can compare total cost of mixed LLM+tool pipelines |

| US-16 | Any user | Navigate battles from a persistent sidebar | I can switch between battles without losing my place |

## Key Workflow: Multi-Step Pipeline Battle (US-1)

1. `pipx install t01-llm-battle && t01-llm-battle serve` → browser opens at `localhost:7878`
2. Click **New Battle**
3. Enter name + judge config (provider, model, rubric)
4. **Add sources**: upload `cases.csv` (10 rows → 10 source items)
5. **Add Fighter A** — "Single model":
   - Step 1: system prompt + GPT-4o-mini + `{"temperature": 0.3}`
6. **Add Fighter B** — "Two-step pipeline":
   - Step 1: "Extract key facts" + Haiku (fast + cheap extraction)
   - Step 2: "Write a concise answer using the facts" + GPT-4o-mini
7. Click **Run**
8. Watch live: step indicators turn green as each step completes
9. Review side-by-side: Fighter B costs more (two calls) — does its score justify it?
10. Read the markdown summary report generated by the judge

## Key Workflow: Human Baseline (US-3)

1. Create a battle with 5 source items
2. Add Fighter A (automated pipeline) + Fighter B (manual)
3. Click **Run** — Fighter A runs automatically; Fighter B shows input boxes
4. Paste or type your answer for each source item
5. Submit manual results → judge scores all fighters together
6. See where human answers rank vs the pipeline
